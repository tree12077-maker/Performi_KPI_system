📌 專案簡介

本元件負責整體後台介面的結構與使用者操作體驗，是 Performi 系統所有模組的共用版型基礎。

🧭

統一導覽體驗

提供一致的 Sidebar 與頁面框架，讓儀表板、設定頁與管理頁能在同一介面語言中運作。

🔐

權限導向顯示

依角色與個別頁面權限，動態控制功能區塊與選單顯示，避免未授權頁面暴露。

🌍

多語與跨裝置

支援中英文切換與桌機 / 手機自適應排版，適合跨廠區與多角色的企業內部環境。

🎯 設計目的

後台版型不只是畫面容器，而是整合導覽、權限與語系能力的核心介面層。

  • 統一導航體驗：所有頁面使用一致的側邊欄與版面結構
  • 角色權限管理：支援 RBAC + Page-level Control 雙層模式
  • 多語系支援：中英文切換，文字排版可隨語系動態調整
  • 響應式設計：桌機與手機皆保有完整可用性
  • 模組化擴充：新功能頁面可快速接入既有架構

🧩 核心功能

從登入、載入、權限判斷到畫面呈現，完整封裝為可重用的後台框架。

  • 使用者資料載入：自動讀取姓名、Email、部門、職位與角色
  • 動態選單：General / Team / Admin 三層導覽結構
  • 多語系支援：LanguageProvider + LanguageSwitcher
  • 響應式 Sidebar：Desktop 完整顯示，Mobile 可收合
  • 使用者選單：顯示身份、管理員標記與安全登出機制

🔐 權限控制邏輯

採用「角色 + 個別頁面權限」雙層控制模型，兼顧簡潔性與擴充性。

Role-based
admin
可看到全部頁面與管理功能。
Role-based
管理職
可看到 Team 區塊與團隊管理相關功能。
Page-level
一般使用者
僅可看到一般功能，部分頁面需額外檢查 PageAccessConfig。
  • 若非 admin，系統會額外查詢 PageAccessConfig
  • 目前控管頁面包含：BatchKpiUpdate、ProductionEfficiency
  • 只有設定為啟用狀態時，使用者才會看到對應功能頁

🚀 適用場景

適合需要多角色管理、權限分流與數據導覽的企業內部系統。

  • KPI 管理系統
  • 部門績效平台
  • 人才 / 組織管理後台
  • QA / 製造數據平台
  • 多角色企業內部管理系統

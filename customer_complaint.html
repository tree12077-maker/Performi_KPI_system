專案背景與挑戰

Before

人工 Excel 管理模式

  • 案件資料分散在不同 Excel 檔案或版本中。
  • 分責與狀態更新仰賴人工維護，容易延遲或遺漏。
  • 主管若要掌握最新進度，通常需要再請同仁手動彙整。
  • 跨部門對資訊的認知不同步，追蹤成本高。
After

Lists + BI 自動化管理模式

  • 客訴案件統一進入 Lists，建立標準欄位與案件主檔。
  • 責任單位、負責窗口、狀態與進度以同一份資料同步管理。
  • Power BI 每日自動刷新，主管可直接查看最新案件。
  • 跨部門改以同一份數據對焦，提升溝通效率與透明度。

我做了什麼

流程重整

盤點原有 Excel 管理方式，重新定義案件欄位、分責規則、追蹤節點與主管需要的管理視角，先把流程標準化，再進行系統化。

資料架構設計

以 Microsoft Lists 建立案件主資料，統一管理責任單位、負責窗口、案件狀態、逾期追蹤等關鍵欄位，降低資料分散與版本不一致問題。

報表與管理視覺化

串接 Power BI 建立每日更新儀表板，讓主管可快速掌握案件量、分責分布、處理進度與逾期風險，提升決策速度。

使用工具

Power BI Dashboard

Dashboard 提供全球客訴分布、失效率趨勢、案件狀態與保固分類分析，讓主管可快速掌握整體異常概況與責任分布。

優化時程

Phase 1

盤點現況與痛點

整理人工流程，釐清案件欄位、分責邏輯、追蹤節點與主管需求，確認主要痛點為人工耗時、資料分散與更新不即時。

Phase 2

建立標準化欄位

將原本 Excel 欄位重新整理，統一命名與定義，作為 Lists 主檔與後續報表串接的基礎。

Phase 3

建置 Lists 主資料

將客訴案件集中進入 Microsoft Lists，形成單一主資料來源，降低多版本檔案流轉風險。

Phase 4

導入 BI 儀表板

串接 Power BI，建立每日自動更新的案件分責、進度追蹤與逾期風險可視化頁面，支援主管日常管理。
